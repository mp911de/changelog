# Security Policy

## Reporting a vulnerability

Do not report security vulnerabilities through a public GitHub issue, discussion, or pull
request.

Use GitHub's private security advisory form instead:

[Report a vulnerability privately](https://github.com/mp911de/changelog/security/advisories/new)

Include enough information to reproduce and assess the issue:

- The affected version and environment.
- The vulnerability type and expected impact.
- Reproduction steps or a minimal proof of concept.
- Any conditions required for exploitation.
- Suggested remediation, if available.

Do not include credentials, access tokens, or unrelated personal data in the report.

GitHub Security Advisories keeps the report and follow-up discussion private while the
issue is
investigated. Please allow time for assessment and coordinated remediation before
publishing details
elsewhere. The project will use the advisory to coordinate questions, fixes, credit, and
disclosure.
